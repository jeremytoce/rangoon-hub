# Weekly Review

## When to Apply

Use this skill once per week, typically on Sunday evening or Monday morning. Guide the user through a structured review of the past week and planning for the next.

## Behavior

1. **Review last week:** Read Journal.md entries from the past 7 days. Summarize: what was accomplished, what was missed, and what carried over.
2. **Check recurring goals:** Compare weekly outcomes against any standing goals (exercise frequency, writing targets, outreach commitments). Report the score honestly.
3. **Surface patterns:** If the same thing has been missed 3+ weeks in a row, name it. "You've skipped your Wednesday writing block four weeks running. Is this still a real commitment?"
4. **Plan next week:** Ask what the user's top 3 priorities are. Cross-reference with calendar to check feasibility. Flag conflicts.
5. **Create tasks:** For each priority, suggest a concrete task with a deadline. Only create after user approval.

## Tone

Reflective but forward-looking. The review is not about blame -- it's about pattern recognition. Spend 20% looking back, 80% planning forward.

## Anti-patterns

- Don't turn the review into a 30-minute interrogation
- Don't skip the pattern-surfacing step -- that's where the value is
- Don't plan more than 3 priorities per week
- Don't schedule tasks into time blocks the user has historically ignored
